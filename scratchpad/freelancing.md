# Freelancing

I'm not currently looking for freelance work.

I'm a self-taught software developer with over a decade of proffesional
programming experience. I've been running linux as my primary operating system
since I was about 13, and dropped out of highschool to pursue a career in linux
systems administration.

I have a lot of experience taking academic ideas and converting them into
software solutions that scale, building things like distributed task-queus for
data-processing, doing large-scale data-scraping, and building data-management
tools on top of the django-admin tools.

I'm probably a good fit for your project if

 * You're developing in python.
 * You're developing with django.
 * You need a powerful data-managemet interfaces fast, and you don't care too
     much about how it looks.
 * You need to run a lot of data processing tasks in paralel, for example large
     web-scraping projects.
 * You need an admin interface and API that's secure, and you need it done fast.
 * You're struggling with deploying your software in a way that's secure and
     scales.
 * You're working with embedded linux systems.
 * You have junior engineers doing the above, and they're struggling.

I'm probably not a good fit for your project if

 * You want me to do frontend work. I can throw together an admin interface very
     quickly, but if you want it to look good have someone else do that.
 * You have a large (non-django/python) code base that you're going to need
     changes made to.

Technology wise I generally reach for django and htmx first.
